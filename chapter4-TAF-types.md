# ISTQB CTAL-TAE - Chapter 4: Test Automation Frameworks

## What is a Test Automation Framework (TAF)?
- A set of rules, tools and guidelines for automated testing
- Provides structure, reusability and maintainability
- Reduces effort and cost of test automation

## Types of TAF

### 1. Linear Framework
- Record and playback approach
- No reusability
- Hard to maintain
- Good for small projects only

### 2. Modular Framework
- Tests divided into small independent modules
- Each module tests a specific functionality
- High reusability across test cases
- Easy to maintain

### 3. Library Architecture Framework
- Extension of modular framework
- Common functions grouped into libraries
- Shared across multiple test scripts
- Better reusability than modular

### 4. Data Driven Framework
- Test data separated from test scripts
- Data stored externally (CSV, Excel, JSON, DB)
- Same script runs with different data sets

### 5. Keyword Driven Framework
- Actions represented as keywords in a table
- Keywords coded separately
- Non-technical testers can create tests

### 6. Hybrid Framework
- Combination of two or more frameworks
- Most widely used in industry
- Example: DDT + KDT combined

## Key Quality Attributes of a TAF
- Maintainability
- Reusability
- Portability
- Robustness
- Performance efficiency
