# ISTQB CTAL-TAE Study Notes

## Chapter 3 - Generic Test Automation Architecture (gTAA)

### What is gTAA?
- gTAA = Generic Test Automation Architecture
- Provides a standard structure for building Test Automation Frameworks (TAF)
- Defined by ISTQB as a reference model

### gTAA Layers
1. **Test Generation** - where tests are designed/created
2. **Test Definition** - where test cases and data are specified
3. **Test Execution** - executes tests against the SUT
4. **Test Adaptation** - connects TAF to the SUT (interfaces, protocols)

### Key Terms
- **SUT** = System Under Test
- **TAF** = Test Automation Framework
- **TAS** = Test Automation Solution (TAF + SUT)
- **DDT** = Data Driven Testing
- **KDT** = Keyword Driven Testing

### Important Points
- TAF should be independent of the SUT
- Maintainability and reusability are key quality attributes
- gTAA supports multiple test automation approaches (DDT, KDT, BDD)
